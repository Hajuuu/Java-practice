## 문제
- 각 기능은 진도가 100%일 때 서비스에 반영할 수 있음
- 뒤에 있는 기능이 앞에 있는 기능보다 먼저 개발될 수 있으며 이때 뒤에 있는 기능은 앞의 기능과 함께 배포
- 각 배포마다 몇 개의 기능이 배포되는지 리턴

## 풀이
- 각 작업마다 100퍼센트까지 소요되는 작업 기간을 계산하여 큐에 넣음
- 큐에서 하나씩 빼면서 현재까지 걸린 시간보다 더 적게 걸린 작업이 있다면 그 작업도 함께 꺼낸뒤 개수 증가
- 현재까지의 작업 기간보다 더 오래 걸린 작업이 있다면 answer에 현재까지의 개수를 저장한다. 이후 prev 값을 변경하고 count도 초기화한다.
